# Personalized Webpage Extension

This branch starts only after the tutorial content is accepted.

## Goal

Turn the finished tutorial into a learner-facing page that is easier to scan, revisit, and follow than a document export.

## Minimum Page Blocks

- learner goal summary
- progress path or module list
- key concepts
- worked example
- practice tasks
- source links
- next-step call to action

## Personalization Signals

Use the learner profile to tune:

- reading depth
- example domain
- pacing or milestone labels
- language and tone
- what gets highlighted as must-do first

## Branching Rule

- If the user only wants a single deliverable, stop at markdown plus requested exports.
- If the user wants a page mockup with no reference HTML, a simple static page is enough.
- If the user already has reference HTML or wants a reusable page family, switch to the local `skill-pageforge` workflow for the page phase.

## Deliverable Rule

The webpage is an extension of the tutorial packet, not a replacement for source-backed tutorial content.
