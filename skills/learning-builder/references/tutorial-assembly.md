# Tutorial Assembly Contract

The tutorial should be written from the learner profile outward, not as a linear dump of source notes.

## Required Sections

1. Learner Snapshot
2. Goal and Success Criteria
3. Prerequisites
4. Concept Map
5. Guided Core Lesson
6. Worked Example or Walkthrough
7. Practice Tasks
8. Common Mistakes
9. Recommended Next Step
10. Source Appendix

## Personalization Rules

- Adapt vocabulary and depth to the stated learner level.
- Make time budget visible. If the learner only has three hours, do not generate a semester-length guide.
- Tie examples to the learner's role or real project when possible.
- Convert vague goals into measurable completion targets.
- Preserve source accuracy even when simplifying language.

## Good Tutorial Behaviors

- explain why a concept matters before showing mechanics
- separate must-know from nice-to-know material
- use one concrete example before branching into options
- include checkpoints so the learner can self-assess
- end with a next action instead of a generic conclusion

## Anti-Patterns

- copying source structure instead of teaching structure
- mixing beginner and advanced assumptions without warning
- adding filler sections with no learning value
- hiding the source list at the cost of auditability
- treating export formatting as the tutorial itself
